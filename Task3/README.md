# Task3

Директория для инфраструктурных материалов Kubernetes, настроек масштабирования и подтверждающих артефактов.

## Примечание по настройке memory request

В `deployment.yaml` для тестового приложения `scaletestapp` лимит памяти оставлен равным `30Mi`, как требуется в задании:

```yaml
limits:
  memory: 30Mi
```

Но значение `requests.memory` для локального тестирования HPA было уменьшено до `1Mi`:

```yaml
requests:
  memory: 1Mi
```

Причина в том, что в локальной среде Ubuntu WSL + Minikube доступно ограниченное количество оперативной памяти, а тестовое приложение почти не увеличивает потребление памяти при HTTP-нагрузке от Locust. Если оставить `requests.memory: 30Mi`, HPA показывает низкую утилизацию памяти и не увеличивает количество реплик.

При `requests.memory: 1Mi` HPA может продемонстрировать масштабирование по памяти без чрезмерной нагрузки на локальный компьютер. Это учебная настройка для проверки работы HPA в локальном Minikube.

В production-среде значение `requests.memory` должно подбираться на основе реального профилирования приложения и фактического потребления ресурсов.

Команды для проверки:

```bash
kubectl get hpa
kubectl get pods
kubectl top pods
```
